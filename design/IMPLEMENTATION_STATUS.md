## URL и навигация

### 📝 Изменились паттерны URL

| Прототип | Реализация | Причина |
|---|---|---|
| `/projects/:id/board` (UUID) | `/projects/DEMO/board` (ключ проекта) | UUID неудобен для разработчиков и в адресной строке |
| `/tasks/:id` (UUID) | `/tasks/DEMO-15` (составной ключ) | Ключ задачи — человекочитаемый идентификатор |

Ключ задачи формируется как `{КЛЮЧ_ПРОЕКТА}-{номер}`, например `DEMO-15`, `TT-42`.

## My Dashboard

### 📝 Реализован упрощённый вариант

Прототип предполагал богатый дашборд с 4 stat-картами и 4 секциями задач (In Progress, To Do, Awaiting My Decision, Needs Revision). В реализации — упрощённая версия:

| Прототип | Реализация |
|---|---|
| 4 stat-карты (открытые, ждут Decision, revision, закрыто за неделю) | 1 карта — суммарное число активных задач |
| Баннеры-алерты (красный revision / жёлтый awaiting) | Нет |
| 4 секции задач | 2 колонки: «Assigned to me» и «Reported by me» |
| Decision Process секции | Нет (Decision Process не реализован) |

**Причина:** Decision Process ещё не реализован, поэтому секции, завязанные на него, пропущены. Структура дашборда будет расширена вместе с Decision Process.

## Kanban-доска

### ✅ Базовый канбан работает
- Колонки, drag-and-drop (HTML5 DnD API)
- Подсветка целевой колонки, placeholder «Drop here»
- Оптимистичное обновление
- Фильтры задач
- Кнопка создания задачи

### 📝 Ключевые отличия от прототипа

| Прототип | Реализация | Причина |
|---|---|---|
| Колонки = личный воркфлоу исполнителя (`Assignment.personal_status`) | Колонки = глобальный статус задачи | Мульти-исполнители пока не реализованы; один `assignee_id` |
| Прогресс-бар решений на карточке (`submitted/total leads`) | Нет | Decision Process не реализован |
| Стек аватаров всех исполнителей | Один аватар (assignee) | Один исполнитель в MVP |
| Счётчик комментариев на карточке | Нет | Не реализован |
| Колонки берутся из workflow напрямую | Колонки — отдельная сущность «BoardColumn», которую настраивает admin | Добавили гибкость: один workflow, разные BoardColumn-конфиги |

### 🆕 Новые фичи на борде (не было в прототипе)
- **Live/Paused переключатель** — включает/выключает SSE-поток обновлений. Точка мигает синим в режиме Live.
- **Колонка «Uncategorized»** — серая, пунктирная; задачи, чей статус не смаплен ни в одну колонку.
- **Настройка BoardColumn** в Project Settings → Board: какие статусы попадают в какую колонку.

## Детальная страница задачи

### 📝 Реализован один вид вместо 4 вкладок

Прототип разбивал детальную страницу на 4 вкладки (Overview, Solutions, Comments, History). В реализации — один вертикальный скролл без вкладок.

**Что реализовано:**
- Заголовок с приоритетом, типом, текущим статусом
- Кнопки переходов по воркфлоу
- Описание
- Список дочерних задач (если есть)
- Блок исполнителя с возможностью назначить/сменить
- Мета-информация (приоритет, тип, репортер, дедлайн, родитель, дата создания)
- Комментарии

**Что не реализовано:**
- ❌ Вкладка Solutions (Decision Process)
- ❌ Вкладка History (аудит-лог)
- ❌ Decision Criteria
- ❌ Assignees table с ролями и personal status
- ❌ Статусы Decision Process (awaiting_decision, in_revision, decided и их визуализация)

### 📝 Отличия в деталях

| Прототип | Реализация | Причина |
|---|---|---|
| Sheet справа (460px) — только из Board/Backlog | Sheet из Board/Backlog + отдельная страница `/tasks/:key` | Удобнее прямая ссылка на задачу |
| Переходы по воркфлоу — кнопки в Overview | Кнопки видны прямо в теле страницы | — |
| Обратный переход — без предупреждения | Обратный переход требует подтверждения (жёлтый блок) | Добавлена защита от случайного отката |

### 🆕 Новые фичи
- **Debug-секция** (свёрнутая) в конце — JSON-дамп задачи и комментариев. Для разработки, дизайн не нужен.

## Цветовая система

Прототип и реализация используют одну и ту же систему oklch-токенов. Маппинг:

| Прототип (`var(--...)`) | Реализация (`--...` в `src/index.css`) |
|---|---|
| `--bg` | `--background` |
| `--bg-card` | `--card` |
| `--bg-muted` | `--muted` |
| `--fg` | `--foreground` |
| `--fg-muted` | `--muted-foreground` |
| `--primary` | `--primary` |
| `--border` | `--border` |
| `--sidebar-bg` | `--sidebar` |

Цвета приоритетов (захардкожены в коде, не CSS-переменные):

| Приоритет | Цвет |
|---|---|
| low | `oklch(0.65 0.08 240)` |
| medium | `oklch(0.55 0.14 200)` |
| high | `oklch(0.60 0.18 55)` |
| critical | `oklch(0.55 0.22 25)` |

Цвета типов задач (тоже захардкожены):

| Тип | Цвет |
|---|---|
| task | `#6366f1` (индиго) |
| bug | `#ef4444` (красный) |
| story | `#10b981` (зелёный) |
| epic | `#f59e0b` (янтарный) |
| decision | `#8b5cf6` (фиолетовый) |

**Тёмная тема:** CSS-переменные готовы, переключатель не реализован. Когда придёт время — добавить `<button>` в Header, который вешает класс `.dark` на `<html>`.

## Технический контекст для новых экранов

- **Стек:** React 19 + TypeScript + Tailwind v4 + shadcn/ui. Компоненты `shadcn` добавляются через `npx shadcn@latest add <name>`.
- **Серверный стейт:** TanStack Query. Новые экраны = новые хуки в `features/*/api.ts`.
- **Роутинг:** React Router v7, файл `src/app/router.tsx`.
- **Дизайн-ветки:** ветки `design/...` дизайнер пушит напрямую в `main` (без PR, по договорённости в `CLAUDE.md`).
