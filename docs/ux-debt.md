# UX-долг (фронтенд)

Открытые UX-проблемы, замеченные при работе с фронтендом, но не блокирующие текущий этап. Закрытые пункты — удалять, история есть в git.

> Для бэкенд-долга — см. [tech-debt.md](./tech-debt.md).

---

## 1. Нет отображения названия текущего проекта

Когда проект открыт, в шапке и сайдбаре везде написано «TaskTrack» — название самого сервиса. Пользователь не понимает, в каком проекте он находится.

**Варианты:** показать `project.name` в шапке (breadcrumb), выделить активный проект в сайдбаре (жирный + другой цвет), или то и другое.

**Когда делать:** доводка Этапа 8 (доводка + Alembic).

---

## 2. Панель детали задачи блокирует работу с бордой

Сейчас `<Sheet>` открывается поверх борды с полупрозрачным оверлеем — пока деталь открыта, карточки нельзя перетащить.

**Варианты:**
- Split-view: борда сужается до ~60%, деталь занимает ~40% справа без оверлея (как в Linear);
- Persistent drawer без backdrop, кликабельный контент за ним остаётся активным;
- Две панели в CSS grid, деталь не закрывает борду.

Технически требует отказа от `<Sheet>` (использует `position:fixed` + backdrop) в пользу layout-уровневого решения.

**Когда делать:** post-MVP. В Этапе 7 пройти S2/S3 удалось без split-view: реальный флоу (lead подаёт Solution → переключение пользователя → DM выносит Decision) полностью совершается через одну открытую панель TaskDetail без обращения к борде.

## 4. Backlog как единственный способ для DM найти свои задачи

Менеджер/decision-maker без Assignment'а не видит свои задачи на kanban-борде (там personal workflow). Сейчас единственная точка входа — вкладка «Backlog» с фильтром «Awaiting my decision».

**Варианты:**
- Бейдж со счётчиком в сайдбаре «Awaiting my decision» рядом с Backlog;
- Глобальный «Inbox» вверху сидбара со списком задач, требующих внимания (DM ждёт Decision, lead получил revision_comment);
- Уведомления (есть в плане Этапа 8).

**Когда делать:** Этап 8 — когда появятся `Notification` и in-app badge.
