# ADR-011: Epic Tree View — древовидный список задач по эпикам

**Статус:** Принято  
**Дата:** 2026-05-09

## Контекст

Kanban и Backlog дают плоский/колоночный взгляд на задачи проекта, но не показывают иерархию «эпик → задачи → подзадачи». Нужен отдельный тип представления (View) для работы с деревом задач.

## Решение

Добавлен новый тип View `epic_tree` (в `ViewType` enum). Компонент `EpicTreeView` реализует:

**Структура данных:**
- Дерево строится на клиенте из плоского списка задач проекта через `flattenVisible(tasks, expanded)`
- Сортировка: эпики выше остальных задач, внутри группы — по ключу (numerically)
- Раскрытие/схлопывание через `Set<string>` expanded; эпики раскрыты по умолчанию и при появлении новых

**UX:**
- Строка: отступ по глубине (20px/уровень) + кнопка-экспандер + иконка типа задачи + ключ-ссылка + заголовок
- Клик на ключ задачи (`<Link>`) → полная страница задачи (`/tasks/:key`)
- Клик на строку (кроме ключа и экспандера) → открывает `TaskDetail` как правую панель (аналогично Backlog/Board)
- Тулбар с кнопкой «Create task»
- Колонки: Task / Status / Assignee (фиксированная ширина)
- Статус показывается цветным бейджем по категории (initial / intermediate / final)
- SSE-подписка на изменения задач проекта в реальном времени

**Где используется:**
- Создаётся как обычный View через Settings → Views; доступен в сайдбаре наравне с Board и Backlog
- Иконка в сайдбаре: `Network` из lucide-react

## Последствия

- Иерархия видна только до глубины, которую пользователь явно раскрыл; нет поиска/фильтрации (v2)
- Статусная карта (`statusMap`) строится по всем воркфлоу проекта — работает корректно при мульти-воркфлоу
- Drag-to-reorder задач в дереве не реализован (отложено)
