# ADR-005: Сборник продуктовых решений

**Статус:** Принято  
**Дата:** 2026-04-25/26

Решения, принятые в ходе подготовки к проектированию. Источники: `08-pre-design-tasks.md` блоки 7–8, диалог с владельцем продукта.

---

## Due date при мульти-исполнителях

Один общий due date на задачу, единый для всех исполнителей. Персональные due date — v2.

## Учёт времени

Логирование времени — на исполнителя. Задача показывает сумму и разбивку по исполнителям. Сравнение оценка/факт — по каждому исполнителю отдельно.

## Эпики и Decision Process

Эпики могут содержать задачи с Decision Process. Прогресс эпика = (закрытых задач) / (всего задач). Decision Process не меняет логику расчёта прогресса.

## Импорт/экспорт

CSV-экспорт задач — с первого дня (🟢 MVP). Импорт из Jira/Trello/GitHub — не делаем (🔴). CSV-формат описан в `15-api.md`.

## Шаблоны задач

Только клонирование существующей задачи. Выделенная система шаблонов с предзаполненными полями — v2.

## Watcher-механика

Автоматические watchers при создании Assignment: все исполнители (lead, reviewer, consultant) + автор задачи (reporter). Остальные подписываются вручную.

## Аналитика Decision Process

В MVP — только на уровне задачи: время от `awaiting_decision` до `decided`, количество revision-циклов. Агрегированные отчёты по Decision Process — v2.

## Видимость проектов

Три уровня: `public` (все в инстансе), `restricted` (явно добавленные пользователи/группы), `private` (только владелец и администраторы). Видимость задачи внутри проекта наследуется от проекта (нет отдельной видимости на уровне задачи в MVP).

## Группы пользователей

Плоские (без вложенности). Один пользователь может состоять в нескольких группах.

## Связи между проектами

В MVP только тип `related` (симметричная, без иерархии). Иерархические связи (программа → проект) — v2. Кросс-проектные связи задач допустимы при наличии read-доступа к обоим проектам.

## Переоткрытие задачи

Может инициировать: Manager, Admin, **и автор задачи (Reporter) — только своей задачи** (`task.reporter_id == current_user.id`).

## Делегирование Decision-maker

Не реализуется в MVP. В задаче хранится `decision_maker_id`; смена через `PATCH /tasks/{id}/decision-maker` — v2.

## Платформа

Web-only, mobile-responsive. PWA и нативные мобильные приложения — v2.

## Браузеры

Chrome ≥ 120, Firefox ≥ 120, Safari ≥ 17.

## Язык интерфейса

Только русский в v1. Код готовится к i18n (строки не хардкодятся в шаблонах).
