# ADR-010: MCP-сервер для агентов — встроенный SSE-транспорт

**Статус:** Принято  
**Дата:** 2026-05-06

## Контекст

Для работы AI-агентов (исполнители задач, PM-агенты, архитектурные/ревью-агенты) нужен программный доступ к TaskTrack. Рассматривались варианты:

1. Агенты используют REST API напрямую (Bearer-токен Keycloak) — требует, чтобы каждый агент умел получать JWT, усложняет auth-flow.
2. Отдельный MCP-сервис — дополнительный Docker-контейнер, отдельный порт, отдельный Traefik-маршрут.
3. MCP-сервер встроен в существующий FastAPI-сервис — один процесс, один порт.

## Решение

MCP-сервер реализован как `app/mcp/` — пакет внутри основного приложения. Монтируется в FastAPI через `app.mount("/mcp", mcp.sse_app())`. Транспорт — исключительно HTTP+SSE (агенты всегда подключаются по сети).

**Библиотека:** `fastmcp>=2.0` — обёртка над официальным `mcp` SDK. Выбрана за минимальный boilerplate и готовый SSE-транспорт.

**Сервисный слой используется напрямую** — MCP-инструменты вызывают `task_service`, `project_service` и т.д. без HTTP-клиента. Сессии открываются через `SessionLocal()` (не через FastAPI dep-injection).

**Аутентификация агентов** — не через Keycloak. Вместо этого: статичные пользователи БД + API-ключи:
- Для нескольких агентов: `MCP_AGENTS=key1:uuid1,key2:uuid2` — каждый ключ маппится на отдельного User в БД.
- Для dev-режима: `MCP_AGENT_USER_ID=<uuid>` — один пользователь, ключ не требуется.

**Инструменты (13 штук):** `list_projects`, `get_project`, `list_tasks`, `get_task`, `get_task_by_key`, `list_my_tasks`, `create_task`, `update_task`, `transition_task_status`, `list_comments`, `add_comment`, `list_workflows`, `get_workflow`.

Управление настройками проекта и воркфлоу через MCP не предусмотрено — только чтение.

## Последствия

- Один сервис, один порт — деплой не усложняется.
- Агенты подключаются через `.mcp.json` с `"type": "sse"` и `url: "https://tasktrack.busypage.ru/mcp/sse"`.
- Разные агенты видят разного автора задач/комментариев — идентичность сохраняется в истории.
- **FR-MCP-001 (отложено):** разные наборы инструментов для разных ролей агентов — через отдельные ASGI-эндпоинты (`/mcp/pm`, `/mcp/executor`) над общим кодом `tools/`.
- Нет поддержки stdio-транспорта — по требованию (агенты только сетевые).
