# ADR-002: Технологический стек и инфраструктура

**Статус:** Принято  
**Дата:** 2026-04-26

## Контекст

Исследовательский запуск, до 50 пользователей, небольшая команда. Нужно выбрать стек, который позволит быстро стартовать, легко дебажить сложную бизнес-логику (Decision Process state machine) и запускаться как с Docker, так и локально.

## Решение

| Компонент | Решение | Причина |
|-----------|---------|---------|
| Язык | Python | Зафиксирован командой |
| Фреймворк | FastAPI | Зафиксирован командой |
| База данных | PostgreSQL 16 | FTS, транзакции, JSON, jsonb |
| Auth | Keycloak | Уже развёрнут в инфре; снимает SSO, управление пользователями |
| Реверс-прокси | Traefik | Уже в инфре |
| Контейнеризация | Docker Compose | Single-server MVP |
| Очередь / кэш | **Отсутствует** | Redis и Celery не нужны в MVP (см. ниже) |
| Email | **Отсутствует** | Убран полностью (см. ADR-003) |
| Фоновые задачи | FastAPI `BackgroundTasks` | Достаточно при отсутствии email |
| Scheduled задачи | APScheduler (in-process) | Напоминания decision-maker'у; не масштабируется горизонтально — ок для MVP |
| Мониторинг | Netdata + Dozzle | Легковесный стек: системные метрики + логи контейнеров |
| Фронтенд | React 19 + TypeScript + Vite (SPA) | Зафиксирован командой |
| UI-компоненты | shadcn/ui + Tailwind CSS v4 | Светлая и тёмная тема через CSS-переменные |
| Состояние сервера | TanStack Query | Кэш и синхронизация запросов к REST API |
| Формы | React Hook Form + Zod | Валидация схем на клиенте |

**Почему Redis и Celery не нужны:**
- Сессии → Keycloak (JWT), Redis не нужен
- Email → убран (ADR-003), очередь не нужна
- APScheduler закрывает scheduled напоминания без зависимостей

**Локальный запуск без Docker обязателен.** Конфигурация через `.env`. При `AUTH_STUB=true` — middleware-заглушка вместо Keycloak (фиксированный пользователь).

**Docker Compose сервисы:** `api` + `db` + `keycloak` + `traefik` + `netdata` + `dozzle`. Все с healthcheck.

## Последствия

- APScheduler перестаёт работать корректно при нескольких инстансах `api` → при горизонтальном масштабировании потребует замены на Redis + Celery Beat.
- Мониторинг: Netdata даёт системные метрики (CPU/RAM/диск/сеть), Dozzle — real-time просмотр логов контейнеров без ELK-стека.
- Фронтенд собирается Vite в статику и отдаётся через Traefik (или отдельный nginx-контейнер). В dev-режиме Vite dev server проксирует запросы к FastAPI.
- TanStack Query управляет кэшем ответов API — инвалидация кэша при мутациях обязательна, особенно при переходах `global_status` задачи.
