# 03. Пользовательские истории — Индекс

Истории переработаны и разбиты по функциональным группам. Каждая группа — отдельный файл в директории `docs/stories/`. Пометки 🟢/🟡/🔴 (из `09-mvp.md`) проставлены к каждой истории в соответствующих файлах.

Для 🟢-историй в файлах указаны Acceptance Criteria (AC).

## Краткое описание групп

### core.md — Ядро системы
Базовые сценарии: вход в систему, работа с проектами и задачами, назначение исполнителей, управление воркфлоу и резолюциями. Здесь сосредоточены истории классического таск-трекера без мульти-исполнительской специфики.

### collaboration.md — Коммуникация и совместная работа
Всё, что связано с коммуникацией внутри задачи: комментарии, упоминания, уведомления (in-app и email), история изменений, учёт времени.

### planning.md — Планирование и управление
Инструменты для планирования и навигации: эпики, подзадачи, метки, кастомные поля, поиск и фильтрация, Kanban-доски, спринты, аналитика, интеграции.

### access.md — Доступ, видимость и группы
Управление правами доступа, группами пользователей, видимостью проектов (public/restricted/private), связями между проектами.

### decision-process.md — Decision Process и мульти-исполнители
Ключевая механика продукта: настройка DecisionCriteria, независимые воркфлоу исполнителей, подача и отзыв Solution, цикл доработки (revision), вынесение Decision. Канонический источник историй по мульти-назначению.
