# Таск-трекер с поддержкой мульти-исполнителей

Документация продукта на ранней стадии: пользовательские потребности, роли и сценарии. На этом этапе нет проектирования модели данных и архитектуры — только продуктовая часть.

## Ключевая особенность продукта

В отличие от классических таск-трекеров (Jira, Linear, YouTrack), здесь **одна задача может выполняться несколькими исполнителями параллельно**, причём каждый исполнитель ведёт задачу через **свой персональный воркфлоу** независимо от других. Вместо автоматического закрытия при завершении — запускается **Decision Process**: каждый исполнитель подаёт свой Solution, decision-maker выносит Decision — выбирает один или несколько Solution как итог. Подробнее — в [07-decision-process.md](./07-decision-process.md).
