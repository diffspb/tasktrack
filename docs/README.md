# Таск-трекер с поддержкой мульти-исполнителей

Документация продукта на ранней стадии: пользовательские потребности, роли и сценарии. На этом этапе нет проектирования модели данных и архитектуры — только продуктовая часть.

## Ключевая особенность продукта

В отличие от классических таск-трекеров (Jira, Linear, YouTrack), здесь **одна задача может выполняться несколькими исполнителями параллельно**, причём каждый исполнитель ведёт задачу через **свой персональный воркфлоу** независимо от других. Закрытие задачи в целом происходит на основе резолюций всех исполнителей по настраиваемым правилам.

## Как читать

1. Начни с [01-needs.md](./01-needs.md) — это контекст «зачем».
2. [02-roles.md](./02-roles.md) — словарь ролей, на который опираются истории.
3. [03-user-stories.md](./03-user-stories.md) — основной массив историй, сгруппированный по доменам.
4. [04-multi-assignee.md](./04-multi-assignee.md) — самая важная часть: содержит продуктовые вопросы, ответы на которые определят модель данных.
5. [05-often-missed.md](./05-often-missed.md) — чек-лист того, что стоит заложить заранее.
6. [06-next-steps.md](./06-next-steps.md) — что делать дальше.

## Статус документа

🟡 Черновик. Истории требуют ревью и приоритизации. Открытые продуктовые вопросы (см. файл 04) должны быть решены до начала проектирования модели данных.
