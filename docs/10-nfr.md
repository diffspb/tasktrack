# 10. Нефункциональные требования (NFR)

Контекст: исследовательский запуск, небольшой масштаб. Проектируем с запасом x5, не x100.

---

## Масштаб

| Параметр | MVP / исследование | Целевой (год 1) |
|---|---|---|
| Пользователей | до 50 | до 300 |
| Проектов | 5–10 | до 50 |
| Задач в системе | до 2 000 | до 30 000 |
| Одновременных сессий | до 20 | до 100 |

---

## Производительность

- Открытие задачи / списка задач: < 500 мс при пиковой нагрузке.
- Поиск по задачам: < 1 с при до 30 000 задач.
- Доступность: 99,0 % (плановые окна обслуживания допустимы).
- Не публичный SaaS — строгих SLA нет.

---

## Аутентификация и безопасность

- Локальный логин: email + пароль (bcrypt).
- Google OAuth — опционально, настраивается при деплое.
- SSO / SAML — v2.
- Сессии: HTTP-only cookie, TTL 30 дней.
- Чувствительные операции (удаление проекта, изменение ролей) — только администратор.
- Аудит-лог: изменения прав, удаления, переход задач в критических статусах.

---

## Данные и бэкапы

- СУБД: PostgreSQL.
- Бэкапы: ежедневно, хранение 30 дней.
- RPO: 24 ч. RTO: 4 ч.
- Soft-delete: задачи и проекты. Жёсткое удаление — только администратор вручную.
- Архивация проекта: данные сохраняются, проект переходит в read-only.
- Политика хранения данных: нет авто-удаления, нет TTL для задач.

---

## Платформа и браузеры

- Только web-приложение, mobile-responsive.
- PWA, нативные мобильные приложения — v2.
- Поддерживаемые браузеры: Chrome ≥ 120, Firefox ≥ 120, Safari ≥ 17.
- Язык интерфейса: только русский в v1. Код готовится к i18n (строки не хардкодятся).

---

## Развёртывание

- Docker Compose (single-server для MVP).
- Хостинг: приватное облако или on-premise (VPS / выделенный сервер).
- Multi-tenant, SaaS-инфраструктура — вне scope.

---

## API

- REST API (JSON).
- MVP: только внутренний API для собственного фронтенда.
- Публичный API с версионированием — v2.
- Вебхуки — v2.
- Идемпотентность API-операций — v2.

---

## Поиск, пагинация, лимиты

- Полнотекстовый поиск: PostgreSQL `tsvector` (достаточно для масштаба MVP).
- Пагинация: offset-based, 50 записей по умолчанию.
- Rate limiting: нет в MVP.
- Вложения: до 20 МБ на файл, до 500 МБ на проект.
- Аватары пользователей: хранятся на локальном диске; отдача через nginx. Лимит — 2 МБ на файл. Для MVP достаточно. S3-совместимое хранилище — v2.

---

## Уведомления

- In-app: все события (назначение, комментарий, смена статуса, упоминание).
- Email: только события Decision Process — задача перешла в `awaiting_decision`, Solution отправлен на доработку, Decision вынесен.
- Email для всех событий, дайджест — v2.
- Push-уведомления (web push) — v2.

---

## Соответствие и приватность

- Персональные данные: только email и имя, хранятся в пределах инстанса.
- Логи не содержат паролей и значений чувствительных полей.
- Нет сторонних аналитических трекеров (без Google Analytics и аналогов).
- GDPR / PDPD — не применимо (внутренний инструмент).
